## 1. Go-public privacy scrub

A go-public safety audit (7 parallel dimensions over the tree + full history)
found no live secrets, but real family PII in docs, git history, and issues.
Remediation, in order:

- **Working-tree scrub** (commit `2aaa892`, "slice A"): pseudonymized the two
  children's names — **Ama → Child A, Rocket → Child B** — scoped so gameplay
  vocabulary ("Rocket launch", "Rocket Launch Countdown") survived. Redacted the
  operator email, home Wi-Fi SSID ("Robisfornia"), home LAN IP (`10.0.0.111`),
  and a real Redfin home address (in two untracked files, deleted). Untracked the
  committed runtime SQLite DB and closed the `data/*` gitignore hole
  (`frontend/data/` escaped it → added `*.db`/`*.sqlite` globs); anchored
  `DEFAULT_DB_PATH` to repo-root.
- **History rewrite**: `git filter-repo --replace-text` over all commits with a
  scoped ruleset (word-bounded names + negative-lookahead to preserve gameplay
  "Rocket"), verified clean both directions on a throwaway mirror.
- **The catch**: on GitHub, the rewrite + force-push was **not sufficient** — old
  commits stayed reachable by SHA (and the docs publish those SHAs), and issue
  edit-history retained the pre-redaction names. **Fix: deleted + recreated the
  repo** and pushed only the clean history. Verified old SHAs now return **422**,
  `gh issue list` is **empty**, and a fresh clone's full history greps clean.
  (Reusable lesson saved to workspace memory `feedback_github_history_rewrite_insufficient`.)
- **Flip**: operator flipped the recreated repo to public.
